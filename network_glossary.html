Networking & Security Glossary

105 A–Z terms — companion to the Network Stack Control Mapping — OSI / TCP/IP Quick Reference Ready Reference.

TermDefinition
ACLAccess Control List — a set of rules on a router or firewall that permits or denies traffic based on IP addresses, ports, or protocols.
ARPAddress Resolution Protocol — resolves IPv4 addresses to MAC addresses on a local network segment. Operates at Layer 2.
AnycastA routing method where a single IP address is assigned to multiple servers. Traffic is routed to the nearest or best-performing instance. Used by DNS root servers and CDNs.
API GatewayA Layer 7 device or service that acts as a single entry point for API traffic. Handles authentication, rate limiting, routing, and request transformation between clients and backend services.
BandwidthThe maximum data transfer capacity of a network link, typically measured in Mbps or Gbps.
BGPBorder Gateway Protocol — the primary routing protocol used to exchange routes between autonomous systems on the internet.
BroadcastA transmission sent to all devices on a network segment. IPv4 uses broadcast addresses; IPv6 replaces broadcast with multicast.
BridgeA Layer 2 device that connects two network segments and forwards frames based on MAC addresses. Largely replaced by switches in modern networks.
CIDRClassless Inter-Domain Routing — a notation system (e.g. /24) that specifies how many bits form the network portion of an IP address.
CSRFCross-Site Request Forgery — an attack that tricks a user's browser into performing unwanted actions on a trusted site where they are authenticated. Mitigated with anti-CSRF tokens and SameSite cookies.
DAIDynamic ARP Inspection — a switch security feature that validates ARP packets against a trusted binding table to prevent ARP poisoning.
DecapsulationThe reverse of encapsulation — the receiving host reads and strips headers layer by layer (L1→L2→L3→L4→L7) until the original application data is delivered.
Default GatewayThe router address a device uses to reach destinations outside its local subnet.
DHCPDynamic Host Configuration Protocol — automatically assigns IP addresses, subnet masks, gateways, and DNS servers to devices joining a network.
DDoSDistributed Denial of Service — an attack that floods a target with traffic from many sources simultaneously, overwhelming its capacity and making it unavailable to legitimate users. Mitigated with rate limiting, CDNs, and scrubbing services.
DLPData Loss Prevention — tools and policies that detect and prevent unauthorised transmission of sensitive data outside the organisation.
DMZDemilitarised Zone — a network segment between the internal network and the internet that hosts public-facing services while isolating them from internal systems.
DNSDomain Name System — translates human-readable domain names (example.com) into IP addresses. Uses UDP port 53 by default.
DNS ServerA server that resolves domain names to IP addresses. Types include recursive resolvers (that query on behalf of clients), authoritative servers (that hold zone records), and forwarding servers.
EDR / XDREndpoint Detection and Response / Extended Detection and Response — tools that monitor endpoints (or multiple telemetry sources) for threats and enable rapid investigation and containment.
EIGRPEnhanced Interior Gateway Routing Protocol — a Cisco-developed advanced distance-vector routing protocol widely used in enterprise networks. Supports fast convergence and unequal-cost load balancing.
802.1XAn IEEE standard for port-based Network Access Control (NAC). Authenticates devices before granting network access using RADIUS and EAP. Used on both wired and wireless networks.
EncapsulationThe process where each protocol layer wraps the previous layer's output with its own header (and at Layer 2, a trailer) before transmission.
EthernetThe dominant wired LAN technology that defines frame format, MAC addressing, and media access at Layer 2.
FCSFrame Check Sequence — a 4-byte error-detection trailer appended to Ethernet frames at Layer 2 to verify data integrity.
FirewallA network security device that filters traffic based on rules. May operate at Layer 3 (packet filter), Layer 4 (stateful), or Layer 7 (application/WAF).
FrameThe Protocol Data Unit (PDU) at Layer 2. Contains a header (source/destination MAC), payload, and FCS trailer.
FTP / SFTPFile Transfer Protocol (plaintext, port 21) / Secure File Transfer Protocol (encrypted over SSH). Used for transferring files between systems.
Full DuplexA communication mode where data can be sent and received simultaneously on the same link, doubling effective throughput.
GatewayA device or system that connects two different networks or protocol domains. The default gateway is typically a router.
Half DuplexA communication mode where data can be sent or received, but not simultaneously. Devices must take turns. Used in older Ethernet hubs and some wireless links.
HubA Layer 1 device that broadcasts incoming signals to all connected ports. No intelligence — does not read MAC addresses. Replaced by switches in modern networks.
HTTP / HTTPSHyperText Transfer Protocol / HTTPS (HTTP over TLS). The foundation of web communication. HTTPS adds encryption, integrity, and authentication.
ICMPInternet Control Message Protocol — used for diagnostics (ping, traceroute) and error reporting at Layer 3. Can be abused for scanning and floods.
IDS / IPSIntrusion Detection System / Intrusion Prevention System — IDS monitors and alerts on suspicious traffic; IPS actively blocks it.
IMAPInternet Message Access Protocol — retrieves email from a server while keeping messages synchronised across multiple devices. Ports 143 (plain) / 993 (TLS).
IP AddressA logical address assigned to a device at Layer 3. IPv4 uses 32-bit addresses; IPv6 uses 128-bit addresses.
IPsecInternet Protocol Security — a suite of protocols that encrypts and authenticates IP traffic. Commonly used in VPNs for site-to-site and remote access tunnels.
JitterVariation in packet arrival times. High jitter degrades real-time applications like VoIP and video conferencing.
LatencyThe time delay between sending a packet and receiving a response. Measured in milliseconds (ms). Lower latency = faster response.
Load BalancerA device or service that distributes incoming traffic across multiple servers to improve performance, availability, and fault tolerance. May operate at Layer 4 or Layer 7.
LDAP / LDAPSLightweight Directory Access Protocol — queries directory services such as Active Directory. LDAPS adds TLS encryption. Ports 389 / 636.
MAC AddressMedia Access Control address — a 48-bit hardware identifier burned into a network interface, used for local network communication at Layer 2.
MalwareMalicious software designed to damage, disrupt, or gain unauthorised access to systems. Includes viruses, worms, trojans, ransomware, spyware, and rootkits.
MFAMulti-Factor Authentication — requires two or more verification factors (something you know, have, or are) to grant access.
MTUMaximum Transmission Unit — the largest packet size (in bytes) that a network link can carry without fragmentation. Ethernet default is 1500 bytes.
MulticastA transmission sent to a specific group of interested receivers rather than all devices (broadcast) or a single device (unicast). Used for streaming, routing updates, and IPv6 neighbour discovery.
NACNetwork Access Control — enforces security policies on devices before granting them access to the network. Often uses 802.1X.
NATNetwork Address Translation — maps private IP addresses to public IP addresses, allowing multiple internal devices to share a single public IP.
NetBIOSNetwork Basic Input/Output System — a legacy Layer 5 API and naming service used for file sharing and name resolution in Windows networks. Largely replaced by DNS and SMB over TCP/IP.
NICNetwork Interface Card — the hardware component that connects a device to a network. Operates at Layer 1 (physical signals) and Layer 2 (MAC addressing).
NTPNetwork Time Protocol — synchronises clocks across networked devices. Critical for log correlation and security forensics. UDP port 123.
OSI ModelOpen Systems Interconnection — a 7-layer conceptual framework for understanding network communication: Physical, Data Link, Network, Transport, Session, Presentation, Application.
OSPFOpen Shortest Path First — a link-state interior routing protocol that calculates the shortest path using Dijkstra's algorithm.
PacketThe Protocol Data Unit (PDU) at Layer 3. Contains an IP header (source/destination IP, TTL) and the transport segment as payload.
PAMPrivileged Access Management — tools and policies that control, monitor, and audit elevated access to critical systems and accounts.
PDUProtocol Data Unit — the name for a unit of data at each OSI layer: Data (L7–5), Segment (L4), Packet (L3), Frame (L2), Bits (L1).
Patch PanelA Layer 1 passive device that provides a central termination point for network cables, allowing organised cross-connections between structured cabling and active equipment.
PhishingA social engineering attack that uses deceptive emails, messages, or websites to trick users into revealing credentials, clicking malicious links, or downloading malware. Variants include spear phishing (targeted) and whaling (executive-targeted).
PingA diagnostic tool that sends ICMP Echo Request packets to a target and measures round-trip time. Used to verify reachability and measure latency.
POP3Post Office Protocol version 3 — retrieves email by downloading messages from the server to a single device. Ports 110 (plain) / 995 (TLS). Less flexible than IMAP.
PortA logical number (0–65535) at Layer 4 that identifies a specific service or application on a host. Well-known ports: 0–1023.
PPTPPoint-to-Point Tunneling Protocol — a legacy VPN protocol operating at Layer 5. Considered insecure due to weak encryption; replaced by L2TP/IPsec, IKEv2, and WireGuard.
Proxy ServerAn intermediary that handles requests between clients and servers, providing caching, filtering, access control, and anonymity.
QoSQuality of Service — techniques that prioritise certain traffic types (e.g. voice, video) over others to ensure performance.
RBACRole-Based Access Control — grants permissions based on a user's organisational role rather than individual identity.
RDPRemote Desktop Protocol — provides graphical remote access to Windows systems. TCP port 3389. A frequent attack target if exposed.
RepeaterA Layer 1 device that regenerates and amplifies electrical or optical signals to extend the range of a network segment. No intelligence — does not inspect data.
RouterA Layer 3 device that forwards packets between different networks based on IP addressing and routing tables.
RPCRemote Procedure Call — a Layer 5 protocol that allows a program to execute a procedure on a remote system as if it were local. Used in distributed systems and Windows environments.
SegmentThe PDU at Layer 4. Contains transport header (source/destination port, sequence number, flags) and application data as payload.
SCTPStream Control Transmission Protocol — a connection-oriented Layer 4 protocol that supports multi-streaming and multi-homing. Combines TCP's reliability with message-boundary preservation. Used in telecom signalling (SS7 over IP) and some WebRTC implementations.
SIEMSecurity Information and Event Management — a platform that aggregates logs from multiple sources, correlates events, and generates security alerts.
SIPSession Initiation Protocol — sets up, manages, and tears down voice and video communication sessions. Ports 5060 (plain) / 5061 (TLS). Foundation of modern VoIP systems.
SMTPSimple Mail Transfer Protocol — sends email between mail servers. Port 25 (relay) / 587 (submission with TLS).
SNMPSimple Network Management Protocol — monitors and manages network devices. SNMPv3 adds encryption and authentication. Ports 161/162.
SocketA communication endpoint defined by the combination of an IP address + port number + protocol (e.g. TCP 192.168.1.1:443).
SSHSecure Shell — provides encrypted remote command-line access and secure file transfer (SFTP/SCP). TCP port 22.
SSL / TLSSecure Sockets Layer / Transport Layer Security — cryptographic protocols that provide encryption, integrity, and authentication for network communication. TLS has replaced SSL.
STPSpanning Tree Protocol — a Layer 2 protocol that prevents broadcast loops in switched networks by blocking redundant paths and maintaining a loop-free topology. Variants include RSTP and MSTP.
SQL InjectionAn application-layer attack where malicious SQL statements are inserted into input fields to manipulate or extract data from a backend database. Mitigated with parameterised queries, input validation, and WAFs.
SubnetA logical subdivision of an IP network. Defined by a subnet mask that separates the network portion from the host portion of an address.
SwitchA Layer 2 device that forwards frames within a local network using MAC address tables. Layer 3 switches can also route.
SyslogA standard protocol for sending log messages to a centralised server for storage, analysis, and alerting. UDP port 514.
TCPTransmission Control Protocol — connection-oriented, reliable Layer 4 protocol. Uses sequence numbers, acknowledgements, and retransmission. 20-byte minimum header.
TCP/IP ModelA 4-layer practical protocol model (Application, Transport, Internet, Network Access) that powers modern networks and the internet.
TelnetA legacy protocol for unencrypted remote command-line access. TCP port 23. Insecure — transmits credentials in plaintext. Replaced by SSH in all modern environments.
TFTPTrivial File Transfer Protocol — a simplified file transfer protocol with no authentication or encryption. UDP port 69. Used for PXE boot, firmware updates, and network device configuration.
ThreatAny potential cause of harm to a system or network — may be human, technical, environmental, or operational.
ThroughputThe actual amount of data successfully transferred over a network in a given time period. Unlike bandwidth (theoretical maximum), throughput reflects real-world performance after accounting for latency, packet loss, protocol overhead, and congestion.
TLS HandshakeThe process where client and server negotiate cipher suites, exchange certificates, and derive session keys before encrypted communication begins.
TracerouteA diagnostic tool that traces the path packets take across a network by sending probes with incrementing TTL values. Shows each intermediate hop and round-trip time. Uses ICMP or UDP.
TransceiverA Layer 1 device that both transmits and receives data signals. In networking, typically refers to SFP/SFP+ modules that convert electrical signals to optical (fibre) signals and vice versa.
TTLTime to Live — a field in the IP header that limits a packet's lifespan by decrementing at each router hop. Prevents infinite routing loops.
UDPUser Datagram Protocol — connectionless, lightweight Layer 4 protocol. No reliability guarantees. 8-byte fixed header. Used for DNS, VoIP, streaming.
uRPFUnicast Reverse Path Forwarding — a router feature that drops packets with spoofed source IP addresses by verifying the return path.
UnicastA one-to-one transmission from a single source to a single destination. The most common communication type on IP networks. Contrast with broadcast (one-to-all), multicast (one-to-group), and anycast (one-to-nearest).
VLANVirtual LAN — logically segments a physical switch into separate broadcast domains, improving security and performance.
VPNVirtual Private Network — creates an encrypted tunnel over an untrusted network (usually the internet) for secure remote access or site-to-site connectivity.
VulnerabilityA weakness in software, configuration, or process that can be exploited by a threat actor. Threat + Vulnerability = Risk.
WAFWeb Application Firewall — inspects HTTP/HTTPS traffic at Layer 7 to block application-layer attacks such as SQL injection, XSS, and CSRF.
Wi-FiWireless networking technology based on IEEE 802.11 standards. Provides LAN connectivity without physical cables. Security depends on WPA3/WPA2 encryption, authentication (802.1X or PSK), and network segmentation.
Wireless APWireless Access Point — a Layer 2 device that bridges wireless clients to a wired network. Broadcasts SSIDs, handles authentication, and manages radio channels and power levels.
XSSCross-Site Scripting — an application-layer attack where malicious scripts are injected into web pages viewed by other users. Types include stored XSS, reflected XSS, and DOM-based XSS. Mitigated with output encoding, CSP headers, and input validation.
Zero TrustA security architecture that assumes no implicit trust. Every access request is verified based on identity, device, context, and policy — regardless of network location (NIST SP 800-207).
