Key metrics & KPIs
What to measure to prove your Secure SDLC is working.
Mean time to remediate (MTTR)

Average time from vulnerability detection to verified fix across all severity levels.

Target: Critical <24h, High <7d, Medium <30d
Vulnerability escape rate

Percentage of vulnerabilities found in production vs. pre-production (shift-left effectiveness).

Target: <10% escaping to production
SAST/DAST coverage

Percentage of codebases and applications actively scanned by automated security tools.

Target: >95% of repos with SAST enabled
Security gate pass rate

Percentage of builds that pass all security gates on first attempt without overrides.

Target: >85% first-pass rate
Mean time to detect (MTTD)

Average time between a security event occurring and being detected by monitoring.

Target: <1 hour for critical events
Dependency currency

Percentage of third-party dependencies that are within one major version of latest.

Target: >80% current, 0 known critical CVEs
Training completion rate

Percentage of engineering staff who completed security training in the past 12 months.

Target: 100% annually
Incident response time

Time from incident declaration to containment achieved, measured per severity.

Target: P1 contained within 4 hours

Glossary
Quick reference for acronyms and terms used throughout this guide.
SASTStatic Application Security Testing — scans source code without execution
DASTDynamic Application Security Testing — tests running applications
IASTInteractive AST — agents inside the app during testing
SCASoftware Composition Analysis — scans third-party dependencies
SBOMSoftware Bill of Materials — inventory of all components
SIEMSecurity Information & Event Management
SOARSecurity Orchestration, Automation & Response
IaCInfrastructure as Code — version-controlled infra definitions
IDS/IPSIntrusion Detection / Prevention System
STRIDESpoofing, Tampering, Repudiation, Info Disclosure, DoS, Elevation
DREADDamage, Reproducibility, Exploitability, Affected users, Discoverability
CISCenter for Internet Security — publishes hardening benchmarks
CVECommon Vulnerabilities & Exposures — unique vuln identifiers
CSPMCloud Security Posture Management
MTTRMean Time to Remediate — avg fix time after detection
MTTDMean Time to Detect — avg time to notice an incident
XDRExtended Detection & Response — cross-layer threat detection
RACIResponsible, Accountable, Consulted, Informed — ownership matrix
ASVSApplication Security Verification Standard (OWASP)
SSDFSecure Software Development Framework (NIST SP 800-218)
